# 약국 데이터 소스 가이드

## 약국 인허가 체계 개요

### 약사 vs 한약사 개국

약사법에 따라 **약사**와 **한약사** 모두 약국을 개설할 수 있다.

- **약사 개국 약국**: 전문의약품 조제/판매 가능 → 건강보험 요양기관 지정 → **요양기관번호(ykiho) 부여**
- **한약사 개국 약국**: 한약 조제/판매 전문. 전문의약품 조제 불가(교차고용 제외) → 요양기관 지정을 받지 않는 경우 있음 → **요양기관번호 미부여 가능**

단, COVID-19 기간 공적마스크 교부를 위해 요양기관번호를 취득한 한약국도 존재한다. 이런 약국은 HIRA 데이터에 포함되지만, 본래 전문의약품 조제를 하지 않는 한약국이므로 성격상 한약사 단독 개국 약국에 해당한다.

이로 인해 **건강보험심사평가원(HIRA) 데이터에는 요양기관번호 없는 약국이 누락**된다. 다만 그 수는 코로나 이후 감소했을 수 있다 (공적마스크 목적 요양기관번호 취득분).

### 교차고용

하나의 약국에 약사와 한약사가 함께 근무하는 형태. 교차고용 약국은:
- 전문의약품 조제(약사) + 한약 조제(한약사) 모두 가능
- 요양기관번호 보유 (약사가 있으므로)
- 인력정보에서 약사 N명 + 한약사 N명으로 확인 가능

### 동물약국

동물용의약품 판매를 위해 별도 인허가를 받은 약국. 일반 약국과 동물약국을 겸하는 경우가 많다.
LOCALDATA에서 별도 서비스ID(`02_03_02_P`)로 관리되며, 일반 약국 데이터(`01_01_06_P`)와 상호명+주소로 매칭하여 동일 약국 여부를 판별한다.

## 데이터 소스별 상세

### 1. LOCALDATA 전국약국표준데이터

| 항목 | 내용 |
|------|------|
| **출처** | 행정안전부 (지방자치단체 기초자료 등록) |
| **시스템** | 지방행정인허가시스템 (localdata.go.kr) |
| **근거법령** | 약사법 제20조~22조 |
| **소관기관** | 보건복지부 약무정책과 (044-202-2495) |
| **갱신주기** | 매일 (2일전 기준 현행화) |
| **다운로드** | `https://www.localdata.go.kr/datafile/each/01_01_06_P_CSV.zip` |
| **인증** | 불필요 (공개 다운로드) |
| **인코딩** | EUC-KR |
| **좌표계** | EPSG:5174 (Bessel 중부원점 TM) → WGS84 변환 필요 |

**포함 범위**: 인허가 기준 전체 약국 (영업중 + 폐업 + 휴업 + 취소)

**데이터 현황 (2026-04-02 기준)**:
| 영업상태 | 건수 |
|----------|------|
| 영업/정상 | 25,766 |
| 폐업 | 44,141 |
| 취소/말소/만료/정지/중지 | 194 |
| 휴업 | 47 |
| **합계** | **70,148** |

**좌표 보유율** (영업중): 25,130건 (97.5%) 보유, 486건 미보유

**주요 컬럼**:
| 컬럼명 | 설명 | 활용 |
|--------|------|------|
| 관리번호 | 인허가 고유 ID (예: `PHMD119733020033...`) | PK |
| 영업상태구분코드 | 01=영업, 02=휴업, 03=폐업, 04=취소 | 필터링 |
| 영업상태명 | 영업/정상, 폐업, 휴업 등 | 표시용 |
| 상세영업상태코드/명 | 세부 상태 (영업중, 사용중 등) | 보조 필터 |
| 사업장명 | 약국 상호 | 매칭/표시 |
| 소재지전체주소 | 지번 주소 | 매칭 |
| 도로명전체주소 | 도로명 주소 | 표시/지오코딩 |
| 소재지전화 | 전화번호 | 표시 |
| 좌표정보x/y(epsg5174) | EPSG:5174 좌표 | 변환 후 지도 표시 |
| 인허가일자 | 개설일 | 표시 |

**핵심 장점**: 요양기관번호 미부여 약국(한약사 단독 개국 등)까지 포함하는 유일한 전수 데이터.

**한계**: 좌표가 EPSG:5174라 변환 필요. 일부 좌표 누락. 요양기관번호/인력정보 없음.

### 3. HIRA 약국정보서비스

| 항목 | 내용 |
|------|------|
| **출처** | 건강보험심사평가원 |
| **data.go.kr ID** | 15001673 |
| **엔드포인트** | `https://apis.data.go.kr/B551182/pharmacyInfoService/getParmacyBasisList` |
| **인증** | `DRUG_API_KEY` (data.go.kr 발급) |
| **갱신주기** | 수시 (실시간 조회) |
| **좌표계** | WGS84 (변환 불필요) |
| **응답형식** | XML / JSON (`_type=json` 파라미터) |

**데이터 현황 (2026-04-04 기준)**: totalCount = **25,689**

**주요 응답 필드**:
| 필드 | 설명 | 활용 |
|------|------|------|
| ykiho | 암호화된 요양기호 (Base64) | PK, 인력정보 조인 키 |
| yadmNm | 요양기관명 | 표시/매칭 |
| clCd / clCdNm | 종별코드/명 (81=약국) | 분류 |
| sidoCdNm / sgguCdNm | 시도/시군구명 | 필터 |
| addr | 주소 | 표시/매칭 |
| telno | 전화번호 | 표시 |
| estbDd | 개설일자 (YYYYMMDD) | 표시 |
| XPos | 경도 (WGS84) | 지도 |
| YPos | 위도 (WGS84) | 지도 |

**페이징**: `pageNo` + `numOfRows` (최대 100). 총 ~257페이지, 약 25분 소요.

**핵심 장점**: WGS84 좌표 직접 제공. ykiho로 인력정보 조인 가능.

**한계**: 요양기관번호 있는 약국만 포함 → 한약사 단독 개국 약국 누락 (LOCALDATA 대비 ~77건 차이). 영업시간 정보 없음.

### 4. 국립중앙의료원 전국 약국 정보 조회 서비스

| 항목 | 내용 |
|------|------|
| **출처** | 국립중앙의료원 |
| **data.go.kr ID** | 15000576 |
| **엔드포인트** | `https://apis.data.go.kr/B552657/ErmctInsttInfoInqireService/getParmacyListInfoInqire` |
| **인증** | `DRUG_API_KEY` |
| **좌표계** | WGS84 |

**데이터 현황**: totalCount = **25,144** (서울만 5,314)

**주요 응답 필드**:
| 필드 | 설명 | 활용 |
|------|------|------|
| hpid | 기관 고유ID | 매칭 키 |
| dutyName | 약국명 | 매칭 |
| dutyAddr | 주소 | 매칭 |
| dutyTel1 | 전화번호 | 표시 |
| dutyTime1s ~ dutyTime7s | 요일별 영업 시작시간 (HHMM) | 영업시간 |
| dutyTime1c ~ dutyTime7c | 요일별 영업 종료시간 (HHMM) | 영업시간 |
| dutyTime8s / dutyTime8c | 공휴일 영업시간 | 영업시간 |
| wgs84Lat / wgs84Lon | WGS84 좌표 | 지도/매칭 |

**영업시간 코드**: 1=월, 2=화, 3=수, 4=목, 5=금, 6=토, 7=일, 8=공휴일.
예: `dutyTime1s=0900, dutyTime1c=2000` → 월요일 09:00~20:00.

**핵심 장점**: 유일하게 영업시간 정보를 제공하는 소스.

**한계**: 당번약국 신고 시스템에 참여하는 약국만 포함 → HIRA 대비 ~500건 적음. 인력정보 없음.

### 5. HIRA 분기별 파일 (전국 병의원 및 약국 현황)

| 항목 | 내용 |
|------|------|
| **출처** | 건강보험심사평가원 보건의료빅데이터개방시스템 |
| **URL** | `https://opendata.hira.or.kr/op/opc/selectOpenData.do?sno=11925` |
| **갱신주기** | 분기 1회 (3월, 6월, 9월, 12월) |
| **다운로드** | DEXT5 컴포넌트 (Playwright 자동화 또는 수동) |
| **형식** | ZIP 내 XLSX 파일 12개 |

**ZIP 내 파일 구성**:
| 번호 | 파일명 | 용도 |
|------|--------|------|
| 1 | 병원정보서비스 | 병원 기본정보 |
| **2** | **약국정보서비스** | **약국 기본정보 (ykiho + 좌표)** |
| 3~11 | 의료기관별상세정보 01~09 | 시설/진료과목/장비 등 |
| **12** | **의료기관별상세정보_10_기타인력정보** | **약사/한약사 인원수** |

**인력정보 파일 구조**:
| 컬럼 | 설명 |
|------|------|
| 암호화요양기호 | ykiho (조인 키) |
| 요양기관명 | 약국명 |
| 기타인력코드 | 071=약사, 072=한약사 |
| 기타인력코드명 | 약사 / 한약사 |
| 기타인력수 | 해당 인력 수 |

**2025.12 기준 인력 현황** (전체 의료기관 포함):
| 인력유형 | 레코드수 |
|----------|---------|
| 약사 | 27,015 |
| 한약사 | 1,102 |
| 물리치료사 | 12,294 |
| 사회복지사 | 2,311 |
| 작업치료사 | 1,078 |

약국만 필터: ykiho로 약국정보서비스 데이터와 조인하여 약국 소속 인력만 추출.

**핵심 장점**: 인력정보(약사/한약사 수)를 일괄로 확보 가능. API 개별 호출(~7시간) 불필요.

**한계**: 분기 갱신이라 최신성 부족 → 서비스에 "인력정보 기준: YYYY년 MM월" 병기 필수.

### 6. HIRA 의료기관별상세정보 API (인력정보 개별 조회)

| **인증** | `DRUG_API_KEY` |

| 필드 | 설명 |

| yadmNm | 요양기관명 |

**신규 등록된 약국의 ykiho만** 선별 호출하여 보완.

## 데이터 소스 간 관계

```
LOCALDATA 약국 (25,766 영업중)     ← 전수 데이터 (기본 레이어)
  │
  ├── 매칭 (상호+주소) ──→ HIRA 약국정보 (25,689)
  │                          ├── ykiho 부여
  │                          ├── WGS84 좌표 보강
  │                          └── 매칭 (상호+주소) ──→ 국립중앙의료원 (25,144)
  │                                                     └── 영업시간 부착
  │
  ├── 미매칭 (~77건) ──→ 요양기관번호 미부여 약국
  │                       (한약사 단독 개국 등)
  │
  ├── 매칭 (상호+주소) ──→ LOCALDATA 동물약국 (13,250 영업중)
  │                          └── is_animal_pharmacy 플래그
  │
  └── ykiho 조인 ──→ HIRA 인력정보 (분기 파일)
                       ├── 약사 수
                       ├── 한약사 수
                       └── 교차고용 판별 (약사>0 AND 한약사>0)
```
